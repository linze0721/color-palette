# Color Palette Generator — Product Requirements Document (v1)

## 1. Summary

A **public, static** web app that generates **5-color palettes** with one click, lets users **copy HEX** per swatch, and **save favorite palettes** to **localStorage**, with a **sidebar** showing the **last 10** saved favorites. **Dark UI**. No login, no backend.

## 2. Goals

- Fast, delightful palette discovery for designers and developers.
- Zero account setup; works offline-capable after first load (static assets).
- Predictable, pleasant colors via **HSL-based** generation (not raw random RGB).

## 3. Non-goals (v1)

- User accounts, cloud sync, or shared URLs with server-side state.
- Image upload / extraction from photos.
- Export to CSS variables, Tailwind config, or design-tool plugins.
- Internationalization beyond English UI (unless trivial copy).
- Production deployment automation (MVP = local/`npm run build` static output).

## 4. Users

- Developers and designers who need quick color inspiration and copy-paste HEX values.

## 5. Core user stories

| ID | Story |
|----|--------|
| US-1 | As a visitor, I click **Generate** and see a new **5-color** palette. |
| US-2 | As a visitor, I click a swatch and **copy its HEX** to the clipboard (with brief feedback). |
| US-3 | As a visitor, I **save the current palette** to favorites and see it in the sidebar. |
| US-4 | As a visitor, I see my **last 10** favorite palettes in the sidebar (newest first). |
| US-5 | As a visitor, I click a favorite in the sidebar to **restore** that palette in the main view. |
| US-6 | As a visitor, I can **remove** a favorite from the sidebar (optional but recommended for UX). |

## 6. Palette generation (HSL)

- Generate **5 colors** per palette.
- **Hue**: spread across the wheel (e.g. base hue + offsets, or evenly spaced hues with small jitter) so palettes feel varied, not muddy duplicates.
- **Saturation**: constrained band (e.g. 45–75%) — configurable constants in code, not user-facing sliders in v1.
- **Lightness**: constrained band (e.g. 35–65% for swatches readable on dark UI); ensure sufficient contrast between swatches where possible.
- **Output**: each color stored and displayed as **HEX** (convert from HSL); optional show of HSL in tooltip is out of scope unless trivial.

## 7. UI / layout (dark)

- **Global**: dark background, light text, accessible focus states.
- **Main area**:
  - Primary CTA: **Generate palette** (or “New palette”).
  - **5 large swatches** in a row/grid; each clickable for copy.
  - **Save to favorites** for the current 5-color set.
- **Sidebar**:
  - Title e.g. “Recent favorites”.
  - List of up to **10** entries; each entry shows **5 mini swatches** + optional timestamp.
  - Click entry → load palette into main area.
  - Remove/delete per entry (recommended).

## 8. Data model (client-only)

### `FavoritePalette`

| Field | Type | Notes |
|-------|------|--------|
| `id` | string | UUID or timestamp-based unique id |
| `colors` | string[5] | HEX strings, e.g. `#3A7BD5` |
| `createdAt` | number | Unix ms |

### localStorage

- Key: e.g. `palette-gen:favorites`
- Value: JSON array of `FavoritePalette`, **newest first**, **max length 10** (drop oldest on save when full).

No other persistence in v1.

## 9. Auth

**None.**

## 10. Tech stack

- **Vite** (latest stable v5.x) + **vanilla JavaScript** (ES modules).
- **HTML** + **CSS** (plain CSS or CSS modules per implementer choice; no React in v1).
- **No** backend, **no** database.
- Clipboard: `navigator.clipboard.writeText` with fallback message if permission denied.

## 11. Quality / acceptance criteria

- [ ] `npm run dev` serves the app; `npm run build` produces static `dist/` suitable for any static host.
- [ ] Generate always produces exactly 5 valid HEX colors using HSL rules above.
- [ ] Click swatch copies HEX; user gets visible confirmation (toast or inline).
- [ ] Save adds current palette to sidebar; list capped at 10; persists across refresh.
- [ ] Clicking sidebar item restores all 5 swatches in main view.
- [ ] UI is dark-themed and usable on mobile (stack swatches / collapsible sidebar acceptable).
